# git ガイドライン

このファイルは、Gitの使用に関するガイドラインを定めています。

## コミット規約

プレフィックスを使用:

- `feat:`
- `fix:`
- `docs:`
- `refactor:`
- `test:`
- `perf:`
- `build:`
- `ci:`
- `chore:`

## コミット方針

- 機能変更とリファクタリングは分離
- 各コミットは単一で一貫性のある変更を表す
- コミット前に `git diff --staged` で変更を確認
- プレフィックス以外は日本語で記述する

## ブランチ戦略

- `master`/`main`/`develop` ブランチに直接コミットを追加しない
- 必ずブランチを切ってから作業する

### ブランチ命名

- 新機能: `feature/<機能名>`
- バグ修正: `fix/<バグ名>`
- 緊急修正: `hotfix/<内容>`
- ドキュメント: `docs/<説明>`
- リファクタリング: `refactor/<内容>`
- 雑務: `chore/<内容>`

## プルリクエスト

- `.github/PULL_REQUEST_TEMPLATE.md` がある場合は必ず使用
- 日本語で記述
