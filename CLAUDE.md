# Claude Code ユーザー設定

このファイルには、全プロジェクト共通で適用される開発ガイドラインを記載しています。

## 基本設定

- 言語: 日本語で回答する
- コードスタイル: シンプルで読みやすいコードを優先

## コマンド実行の原則

- スラッシュコマンド実行時は、必ず該当する定義ファイル（.md）を確認する
- 現状に関わらず、定義された手順を省略しない
- 手順を省略すべき状況だと判断した場合は、先にユーザーに確認する

## コード変更の方針

- 依頼された範囲のみ変更する
- 不要なリファクタリングや「改善」は行わない
- 変更が広範囲に及ぶ場合は事前に確認する

## 命名規則

- コンポーネント: PascalCase
- 関数/変数: camelCase
- 定数: UPPER_SNAKE_CASE
- ファイル: camelCase

## コメント

- 複雑なロジックにのみコメントを書く
- 自明なコードにはコメント不要
- TODO/FIXMEは具体的な内容を記載

## テスト

- コード変更後はテストを実行して確認する
- テストが失敗した場合は修正する

## セキュリティ

- `.env` や認証情報を含むファイルはコミットしない
- 機密情報をコードやコメントに含めない
- APIキーやパスワードはハードコードしない

## Git ワークフロー

### コミット規約

プレフィックスを使用:

- `feat:`
- `fix:`
- `docs:`
- `refactor:`
- `test:`
- `perf:`
- `build:`
- `ci:`
- `chore:`

### コミット方針

- 機能変更とリファクタリングは分離
- 各コミットは単一で一貫性のある変更を表す
- コミット前に `git diff --staged` で変更を確認
- プレフィックス以外は日本語で記述する
- `master`/`main`/`develop` ブランチに直接コミットを追加しない

### ブランチ命名

- 新機能: `feature/<機能名>`
- バグ修正: `fix/<バグ名>`
- 緊急修正: `hotfix/<内容>`
- ドキュメント: `docs/<説明>`
- リファクタリング: `refactor/<内容>`
- 雑務: `chore/<内容>`

### プルリクエスト

- `.github/PULL_REQUEST_TEMPLATE.md` がある場合は必ず使用
- 日本語で記述
- `Closes #123` 形式で関連Issueをリンク
